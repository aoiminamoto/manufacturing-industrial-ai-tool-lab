# Industrial AI Platform Evidence Framework

This folder organizes public-safe evidence of the Manufacturing AI Translation Platform's engineering evolution.

## Evidence Layers

1. [Platform evolution timeline](platform-evolution-timeline.md) - dated milestones tied to public Git history
2. [Evidence index](evidence-index.md) - map of claims to public and private supporting records
3. [Usage and quality metrics template](usage-quality-metrics-template.md) - repeatable, non-sensitive measurement structure
4. [Manager and engineer confirmation template](manager-engineer-confirmation-template.md) - factual third-party validation guide
5. [Deployment and Yokoten evidence template](deployment-yokoten-evidence-template.md) - rollout and reuse evidence guide
6. [Architecture Decision Records](../architecture-decisions/) - documented engineering decisions and tradeoffs
7. [July 17 production-hardening evidence](2026-07-17-production-hardening.md) - sanitized implementation and validation record

## Evidence Integrity Rules

- Do not backdate Git commits or create unsupported milestones.
- Distinguish the date work occurred from the date it was retrospectively documented.
- Link public claims to commits, code, tests, or architecture records.
- Keep private operational evidence outside the public repository.
- Do not publish organization logos, internal screenshots, controlled terminology, user identities, server details, or credentials.
- Use `Not yet measured` when a metric does not have a reliable source.

This framework supports a credible technical-leadership portfolio. It is not a substitute for independent legal or professional evaluation of any specific evidence standard.
